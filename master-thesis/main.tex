\begin{abstract}
This thesis investigates how streaming communication can be integrated into DYNAMOS, a policy-driven middleware for dynamically composed microservice workflows. The original DYNAMOS data path is built around unary request-response transfers, which creates a bottleneck when large result sets must be moved through generated service chains. The thesis first compares candidate communication mechanisms in a controlled Kubernetes pipeline, using throughput, latency, pacing, correctness, recovery, and resource pressure as evaluation dimensions. It then implements three chunk-aware DYNAMOS data paths: chunked unary transfer, intra-job gRPC streaming, and RabbitMQ Streams at the agent boundary. A local validation on bulk \texttt{dataThroughTtp} retrieval shows that all three chunked implementations reduce completion time and peak memory compared with the original classic-unary baseline, with additional checks on \texttt{computeToData} and multi-provider \texttt{dataThroughTtp}. The results show that streaming is not one design choice but a set of architectural tradeoffs between raw transfer efficiency, temporal decoupling, recovery behaviour, and fit with DYNAMOS' policy-driven execution model.

\end{abstract}
